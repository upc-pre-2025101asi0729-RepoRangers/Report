# Capítulo III: Requirements Specification
## 3.1. To-Be Escenario Mapping
- **Primer segmento: Usuarios que desean alquilar bicicletas (arrendatarios)**
- **Segundo segmento: Usuarios que ofrecen sus bicicletas en alquiler (arrendadores)**
## 3.2. User Stories
A continuación se presentan las épicas que describen las funcionalidades clave para la aplicación web.
| Epic ID | Título | Descripcion |
|--------|--------|------|
|EP01 |Implementación de Landing page | Como visitante quiero poder visualizar las diferentes características de la plataforma en su landing page para conocer información y acceder a la aplicación web de la misma
|EP02|Gestion de cuenta|Como usuario quiero crear una cuenta y acceder a la plataforma, para realizar alquileres o publicar mis bicicletas 
|EP03|Gestión de bicicletas para arrendadores|Como arrendador quiero registrar y gestionar mi bicicleta para poder ofrecerla en alquiler
|EP04|Alquiler para usuarios|Como arrendatario quiero encontrar, reservar y pagar mi bicicleta, para tener una forma de transporte alternativa
|EP05|Seguridad y soporte|Como usuario quiero sentirme seguro durante el uso de la plataforma para usarla de forma recurrente

Para asegurar que se desarrollen funcionalidades específicas que el usuario necesita, las siguientes user stories han sido definidas. Cada una de estas historias será implementada y validada durante el desarrollo.  

| ID User Story | Título | Descripción | Criterios de aceptación | ID Epic |
|---------------|--------|-------------|-------------------------|---------|

|US06|Selección de rol|Como usuario, quiero poder elegir si quiero registrarme como arrendador o arrendatario desde la pantalla principal para que la plataforma me dirija a la pantalla de registro correspondiente|Escenario 1: Selección de "Arrendador"<br>Dado que el usuario está en la pantalla principal<br>Cuando hace clic en la opción "Soy Arrendador" <br>Entonces se redirige a la pantalla de registro para arrendadores<br><br>Escenario 2: Selección de "Arrendatario"<br>Dado que el usuario está en la pantalla principal<br>Cuando hace clic en la opción "Soy Arrendatario"<br>Entonces se redirige a la pantalla de registro para arrendatarios.|EP02|

|US08|Registro con cuenta de google|Como usuario, quiero poder registrarme utilizando mi cuenta de Google para facilitar el acceso|Escenario 1: Registro exitoso con cuenta de Google<br>Dado que el usuario se encuentra en la pantalla de Registro<br>Cuando haga click en el boton "Continuar con Google"<br>Y autoriza el acceso a su cuenta<br>Entonces su cuenta se registra automáticamente en la plataforma<br>Y se redirige a la sección principal de la plataforma <br><br>Escenario 2: Usuario cancela el acceso a Google <br>Dado que el usuario hizo click en el boton "Continuar con Google"<br>Cuando cancela el acceso a su cuenta de Google<br>Entonces el registro se cancela<br>Y permanece en la pantalla de registro sin cambios.|EP02|
|US09|Inicio de sesión|Como usuario registrado, quiero poder iniciar sesión con mis credenciales para acceder a mi cuenta y funcionalidades|Escenario 1: Inicio de sesión exitoso <br>Dado que el usuario se encuentra en la pantalla de Inicio de sesión<br>Cuando ingresa las credenciales válidas<br>Entonces puede acceder a su cuenta<br>Y se redirige a la sección principal de la plataforma<br><br> Escenario 2: Credenciales incorrectas<br>Dado que el usuario se encuentra en la pantalla de Inicio de sesión<br>Cuando ingresa una contraseña o correo incorrectos<br>Entonces se mostrará un mensaje indicando que el correo o contraseña son incorrectos.|EP02|
|US10|Edición de datos personales|Como usuario, quiero poder actualizar mi perfil y datos personales para mantener mi información actualizada|Escenario 1: Edición de datos personales<br>Dado que el usuario está en la sección de perfil<br>Cuando realiza algún cambio a sus datos personales como nombre, teléfono o dirección<br>Y hace click en la opción "Guardar"<br>Entonces los cambios se guardan correctamente<br><br>Escenario 2: Validación de campos incorrectos<br>Dado que el usuario está en la sección de perfil<br>Cuando llena un campo con información inválida<br>Y hace click en la opción "Guardar" <br>Entonces se mostrará un mensaje indicando cuales son los campos inválidos<br>Y no se guardan los cambios<br><br>Escenario 3: Usuario no realiza modificaciones<br>Dado que el usuario está en la sección de perfil<br>Cuando hace click en la opción "Guardar" sin haber realizado cambios<br>Entonces se mostrará un mensaje indicando que no se realizaron cambios|EP02|
|US11|Cierre de sesión|Como usuario, quiero cerrar sesión de forma segura para proteger mi cuenta cuando dejo de usar la plataforma|Escenario 1: Cierre de sesión exitoso<br>Dado que el usuario ha iniciado sesión en su cuenta<br>Cuando hace clic en el botón de "Cerrar sesión"<br>Entonces el sistema cierra la sesión del usuario<br>Y lo redirige a la página de inicio de sesión<br><br>Escenario 2: Cierre de sesión automático por inactividad<br>Dado que el usuario ha iniciado sesión en su cuenta<br>Y permanece inactivo durante un período de tiempo<br>Cuando el tiempo de sesión expira<br>Entonces el sistema cierra la sesión automáticamente<br>Y se muestra un mensaje de sesión expirada|EP02|
|US12|Registro de una bicicleta|Como arrendador, quiero registrar una nueva bicicleta con sus datos y fotos para que los arrendatarios puedan verla y alquilarla|Escenario 1: Registro exitoso de una bicicleta<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando ingresa los datos de la bicicleta (marca, modelo, tamaño, estado, precio)<br>Y sube fotos de la bicicleta<br>Y hace clic en "Publicar bicicleta"<br>Entonces muestra un mensaje de éxito<br>Y la bicicleta aparece disponible en la plataforma para alquiler<br><br>Escenario 2: Ingreso de datos incompletos<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando intenta registrar una bicicleta sin llenar campos obligatorios<br>Entonces se muestra un mensaje de error indicando los campos faltantes<br>Y no se permite el registro hasta completar la información.|EP03|
|US13|Edición de información de una bicicleta|Como arrendador, quiero editar la información de mis bicicletas para actualizar detalles|Escenario 1: Edición exitosa<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta<br>Y modifica su información<br>Y guarda los cambios<br>Entonces el sistema muestra un mensaje de éxito<br>Y la bicicleta se actualiza con la nueva información en la plataforma<br><br> Escenario 2: Edición con campos incompletos o inválidos<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando elimina información obligatoria<br>Y intenta guardar los cambios<br>Entonces se muestra un mensaje indicando los datos faltantes<br>Y no permite la actualización hasta completar los campos requeridos|EP03|
|US14|Eliminación de una bicicleta|Como arrendador, quiero eliminar bicicletas que ya no deseo alquilar para mantener mi listado limpio y actualizado|Escenario 1: Eliminación exitosa<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta de su listado<br>Y hace clic en el botón de "Eliminar"<br>Entonces se muestra una confirmación de eliminación<br>Y el arrendador confirma la acción<br>Y la bicicleta desaparece del listado<br><br>Escenario 2: Cancelación de eliminación <br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta de su listado para eliminarla<br>Y se muestra una confirmación de eliminación<br>Y el arrendador decide cancelar<br>Entonces la bicicleta permanece en el listado sin cambios|EP03|
|US15|Historial de alquileres|Como arrendador, quiero visualizar un historial de alquileres de mis bicicletas para conocer la demanda y evaluar ingresos|Escenario 1: Visualización de historial<br>Dado que el arrendador ha iniciado sesión en su cuenta<br>Y ha tenido bicicletas alquiladas anteriormente<br>Cuando accede a la sección "Historial de alquileres"<br>Entonces se muestra una lista con los datos sobre el alquiler de las bicicletas<br><br>Escenario 2: Historial vacío<br>Dado que el arrendador ha iniciado sesión en su cuenta<br>Y ninguna de sus bicicletas ha sido alquilada<br>Cuando accede a la sección "Historial de alquileres"<br>Entonces se muestra el mensaje "Aún no tienes alquileres registrados"|EP03|
|US16|Visualización de estadísticas de uso|Como arrendador, quiero visualizar estadísticas de uso e ingresos generados por mis bicicletas para tomar mejores decisiones sobre mi oferta|Escenario 1: Estadísticas generales<br>Dado que el arrendador ha iniciado sesión en su cuenta<br>Cuando accede a la sección "Estadísticas" <br>Entonces el sistema muestra métricas generales como número de alquileres, ingresos totales y frecuencia de uso<br><br>Escenario 2: Visualización por bicicleta específica <br>Dado que el arrendador se encuentra en la sección "Estadísticas"<br>Cuando selecciona una bicicleta del listado<br>Entonces se muestran estadísticas solo para esa bicicleta|EP03|
|US17|Calificaciones y comentarios de arrendatarios|Como arrendador, quiero ver calificaciones y comentarios de arrendatarios sobre mis bicicletas para mejorar el servicio que ofrezco|Escenario 1: Visualizar comentarios de arrendatarios<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta de su listado<br>Y accede a la opción "Ver calificaciones y comentarios"<br>Entonces se muestra una lista con las calificaciones y comentarios de los arrendatarios <br><br>Escenario 2: Bicicleta sin calificaciones<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta de su listado<br>Y la bicicleta nunca ha sido evaluada<br>Entonces se muestra un mensaje indicando que aún no hay calificaciones disponibles|EP03|
|US18|Configuración de disponibilidad|Como arrendador, quiero establecer días y horas disponibles para cada bicicleta para evitar reservas en momentos no deseados|Escenario 1: Definición de horario disponible<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta de su listado<br>Y configura los días y horarios en los que estará disponible para alquiler<br>Y guarda la información<br>Entonces la bicicleta solo podrá ser alquilada en los horarios establecidos<br><br>Escenario 2: Modificación de la disponibilidad<br>Dado que el arrendador se encuentra en la sección "Mis bicicletas"<br>Cuando selecciona una bicicleta<br>Y edita su configuración de disponibilidad<br>Y guarda los cambios<br>Entonces se actualiza la información|EP03|
|US19|Comunicación con el arrendatario|Como arrendador, quiero poder contactar al arrendatario antes o durante el alquiler para resolver dudas o emergencias|Escenario 1: Contacto antes del alquiler<br>Dado que el arrendador ha publicado una bicicleta para alquiler<br>Y un arrendatario ha mostrado interés<br>Cuando el arrendador accede a la opción "Contactar al arrendatario"<br>Entonces el sistema permite enviar un mensaje con información adicional<br><br>Escenario 2: Emergencia durante el alquiler <br>Dado que el arrendador ha alquilado una bicicleta a un arrendatario<br>Cuando accede a la opción "Enviar alerta urgente"<br>Entonces se envía una notificación prioritaria al arrendatario|EP03|

|US21|Visualización de detalles de bicicletas|Como arrendatario, quiero ver información detallada de cada bicicleta incluyendo reseñas antes de alquilarla para tomar una decisión informada|Escenario 1: Visualización de detalles completos <br>Dado que el arrendatario se encuentra en la sección "Buscar bicicletas"<br>Cuando selecciona una bicicleta de la lista<br>Entonces muestra una página con la información completa de la bicicleta<br><br>Escenario 2: Visualización de calificaciones y comentarios de otros usuarios<br>Dado que el arrendatario se encuentra en la página de detalles de una bicicleta<br>Cuando accede a la opción "Calificaciones y comentarios"<br>Entonces se muestran opiniones de usuarios anteriores|EP04|
|US22|Alquiler de bicicleta|Como arrendatario, quiero poder reservar una bicicleta por un periodo específico para utilizarla según mi necesidad|Escenario 1: Reserva exitosa <br>Dado que el arrendatario desea alquilar una bicicleta<br>Cuando elige el período de alquiler (fecha y duración)<br>Y confirma la reserva<br>Entonces el sistema registra la reserva<br>Y la bicicleta queda bloqueada para otros usuarios en ese período<br><br>Escenario 2: Conflicto de horario<br>Dado que el arrendatario desea alquilar una bicicleta<br>Cuando elige una fecha en la que no está disponible<br>Entonces el sistema muestra el mensaje "La bicicleta no está disponible en ese horario"|EP04|
|US23|Gestión de alquileres|Como arrendatario, quiero ver mis alquileres actuales y pasados para gestionar el uso y control de mis reservas|Escenario 1: Ver alquileres en curso <br>Dado que el arrendatario tiene reservas activas<br>Cuando accede a la sección "Mis alquileres"<br>Entonces se muestran las bicicletas alquiladas actualmente y sus detalles<br><br>Escenario 2: Modificación de una reserva<br>Dado que el arrendatario ha reservado una bicicleta<br>Y accede a la sección "Mis alquileres"<br>Cuando modifica la fecha o duración del alquiler antes de la fecha de inicio<br>Entonces el sistema actualiza la reserva<br>Y muestra una notificación de confirmación<br><br>Escenario 3:Cancelación de una reserva<br>Dado que el arrendatario ha reservado una bicicleta<br>Y accede a la sección "Mis alquileres"<br>Cuando selecciona la opción "Cancelar reserva"<br>Entonces el sistema elimina la reserva|EP04|

|US25|Pago del alquiler|Como arrendatario, quiero poder realizar el pago del alquiler dentro de la plataforma para asegurar la reserva de la bicicleta seleccionada|Escenario 1: Pago exitoso<br>Dado que el arrendatario ha iniciado el proceso de pago<br>Cuando accede a la opción "Realizar pago"<br>Y ingresa los datos de su tarjeta o método de pago<br>Y confirma la transacción<br>Entonces el sistema procesa el pago exitosamente<br>Y muestra una confirmación de la reserva<br><br>Escenario 2: Fallo en el método de pago<br>Dado que el arrendatario ha iniciado el proceso de pago<br>Cuando intenta realizar el pago con una tarjeta expirada o datos incorrectos<br>Entonces el sistema muestra un mensaje de error<br>Y no se procesa la reserva<br><br>Escenario 3: Cancelación de pago<br>Dado que el arrendatario ha iniciado el proceso de pago<br>Cuando presiona la opción "Cancelar" antes de confirmar<br>Entonces el sistema cancela la transacción<br>Y la bicicleta sigue disponible para otros usuarios|EP04|
|US26|Comunicación con el arrendador|Como arrendatario, quiero poder contactar con el arrendador antes y durante el alquiler para coordinar la entrega o resolver dudas|Escenario 1: Contacto antes del alquiler<br>Dado que el arrendatario se encuentra en la página de detalles de una bicicleta<br>Cuando el arrendatario accede a la opción "Contactar al arrendador"<br>Entonces el sistema permite enviar una consulta sobre disponibilidad o condiciones de uso<br><br>Escenario 2: Emergencia durante el alquiler <br>Dado que el arrendatario ha alquilado una bicicleta<br>Cuando accede a la opción "Enviar alerta urgente"<br>Entonces se envía una notificación prioritaria al arrendador|EP04|
